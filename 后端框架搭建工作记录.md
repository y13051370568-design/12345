```python
class Settings(BaseSettings):
    # 数据库配置
    DB_HOST: str = "localhost"
    DB_PORT: int = 3306
    DB_USER: str = "root"
    DB_PASSWORD: str = ""
    DB_NAME: str = "ai4ml_community"

    def database_url(self) -> str:
        return f"mysql+pymysql://{self.DB_USER}:{self.DB_PASSWORD}@{self.DB_HOST}:{self.DB_PORT}/{self.DB_NAME}"
```

```python
engine = create_engine(
```

### 2.3 统一异常处理模块

#### 创建文件：`app/core/exceptions.py`
**文件路径**: `d:\github\12345\app\core\exceptions.py`

**功能说明**:
- 定义业务异常类型体系
- 统一错误响应格式
- 提供友好的用户错误提示
- 防止技术堆栈信息泄露

**异常类型体系**:
```
BusinessException (基类)
├── AuthenticationException     # 认证异常 (401)
├── AuthorizationException      # 权限异常 (403)
├── ResourceNotFoundException   # 资源不存在 (404)
├── DataValidationException     # 数据校验异常 (422)
├── FileUploadException         # 文件上传异常 (400)
├── TaskExecutionException      # 任务执行异常 (500)
├── QuotaExceededException      # 配额超限异常 (429)
└── ConcurrentLimitException    # 并发限制异常 (503)
```

**统一响应格式**:
```json
{
  "success": false,
  "code": "RESOURCE_NOT_FOUND",
  "message": "请求的资源不存在",
  "details": {},
  "path": "/api/tasks/123"
}
```

**全局异常处理器**:
- `business_exception_handler`: 处理业务异常
- `http_exception_handler`: 处理HTTP异常
- `validation_exception_handler`: 处理参数校验异常
- `general_exception_handler`: 处理未捕获异常（兜底）

**使用示例**:
```python
# 抛出业务异常
if not user:
    raise ResourceNotFoundException("用户不存在")

# 抛出数据校验异常
if file_size > MAX_SIZE:
    raise DataValidationException(
        message="文件大小超过限制",
        details={"max_size": MAX_SIZE, "actual_size": file_size}
    )
```

**核心功能实现**:

1. **CORS配置**（支持前端跨域）:
```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],  # 生产环境应配置具体域名
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

2. **请求日志中间件**（记录每个HTTP请求）:
```python
@app.middleware("http")
async def log_requests(request: Request, call_next):
    start_time = time.time()
    response = await call_next(request)
    duration = time.time() - start_time
    log_request(request.method, str(request.url.path), response.status_code, duration)
    return response
```

3. **异常处理器注册**:
```python
app.add_exception_handler(BusinessException, business_exception_handler)
app.add_exception_handler(StarletteHTTPException, http_exception_handler)
app.add_exception_handler(RequestValidationError, validation_exception_handler)
app.add_exception_handler(Exception, general_exception_handler)
```

   - `GET /redoc`: ReDoc API文档

## 九、注意事项与最佳实践

### 9.1 环境变量安全

⚠️ **重要提示**:
- `.env` 文件包含敏感信息，已添加到 `.gitignore`
- 生产环境必须修改 `SECRET_KEY` 和数据库密码
- 不要将 `.env` 文件提交到Git仓库

### 9.2 日志管理

📝 **日志建议**:
- 开发环境设置 `LOG_LEVEL=DEBUG`
- 生产环境设置 `LOG_LEVEL=INFO` 或 `WARNING`
- 定期清理旧日志文件
- 敏感信息（密码、Token）不要记录到日志

### 9.3 异常处理

🔧 **异常处理原则**:
- 业务层抛出具体的业务异常（继承自BusinessException）
- 不要直接抛出通用Exception
- 异常信息要对用户友好，不暴露技术细节
- 关键操作要记录异常日志（exc_info=True）

### 9.4 数据库连接

💾 **数据库最佳实践**:
- 使用依赖注入方式获取数据库会话
- 事务操作使用上下文管理器确保自动提交/回滚
- 长时间操作要注意连接超时
- 生产环境调整连接池参数
