**完成时间**: 2026年4月19日

## 一、功能概述

完整实现了用户端的登录认证功能，包括：

✅ **1. 开发登录认证API接口**
- 用户注册接口
- 用户登录接口
- Token刷新接口
- 获取当前用户信息接口
- 登出接口
- 权限查询接口
- 路由获取接口

✅ **2. 实现JWT/Token生成与验证逻辑**
- JWT访问令牌生成（有效期30分钟）
- JWT刷新令牌生成（有效期7天）
- Token验证与解码
- 密码加密与验证（bcrypt）

✅ **3. 实现不同角色的权限校验**
- 三种用户角色：ZERO_BASIS（零基础用户）、DEVELOPER（开发者）、ADMIN（管理员）
- 基于角色的访问控制（RBAC）
- 细粒度权限验证
- 权限装饰器和依赖注入

✅ **4. 对接前端登录请求**
- 返回用户信息
- 返回访问令牌和刷新令牌
- 返回角色对应的跳转路由
- 返回用户权限列表

## 二、核心文件说明

### 2.1 安全模块 - `app/core/security.py` ⭐

**功能**: JWT Token生成与验证、密码加密工具类

**核心类**:

#### `PasswordHandler` - 密码处理器
```python
# 加密密码
hashed = PasswordHandler.hash_password("plain_password")

# 验证密码
is_valid = PasswordHandler.verify_password("plain_password", hashed)
```

#### `JWTHandler` - JWT处理器
```python
# 生成访问令牌（30分钟有效）
access_token = JWTHandler.create_access_token({
    "user_id": 1,
    "username": "admin",
    "role": "ADMIN"
})

# 生成刷新令牌（7天有效）
refresh_token = JWTHandler.create_refresh_token(data)

# 验证令牌
payload = JWTHandler.verify_token(token, token_type="access")
```

**技术实现**:
- 密码加密：使用 `passlib` 的 `bcrypt` 算法
- JWT签名：使用 `python-jose` 实现
- Token载荷包含：user_id, username, role, exp（过期时间）, iat（签发时间）, type（令牌类型）

### 2.2 认证依赖 - `app/core/auth.py` ⭐

**功能**: JWT验证、用户身份获取、权限校验的依赖注入函数

**核心依赖函数**:

#### 1. `get_current_user` - 获取当前用户
```python
@router.get("/protected")
def protected_route(current_user = Depends(get_current_user)):
    return {"user": current_user.username}
```

#### 2. `require_roles` - 角色权限验证
```python
# 只允许管理员访问
@router.get("/admin/users")
def admin_only(user = Depends(admin_required)):
    return {"message": "Admin area"}

# 只允许开发者和管理员访问
@router.get("/developer/code")
def dev_area(user = Depends(developer_required)):
    return {"message": "Developer area"}
```

#### 3. `require_permissions` - 细粒度权限验证
```python
@router.post("/tasks")
def create_task(user = Depends(require_permissions(["task:create"]))):
    return {"message": "Task created"}
```

**预定义权限依赖**:
- `admin_required`: 仅管理员
- `developer_required`: 开发者或管理员
- `user_or_developer`: 所有已登录用户

## 四、前端对接指南

### 4.1 登录流程

```javascript
// 1. 用户登录
async function login(username, password) {
  const response = await fetch('/api/auth/login', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ username, password })
  });

  const result = await response.json();

  if (result.success) {
    // 2. 保存Token
    localStorage.setItem('access_token', result.data.access_token);
    localStorage.setItem('refresh_token', result.data.refresh_token);
    
    // 3. 保存用户信息
    localStorage.setItem('user', JSON.stringify(result.data.user));
    
    // 4. 保存权限列表
    localStorage.setItem('permissions', JSON.stringify(result.data.permissions));
    
    // 5. 跳转到对应页面
    window.location.href = result.data.redirect_route;
  } else {
    alert('登录失败：' + result.message);
  }
}
```

### 4.2 携带Token请求

```javascript
// 方式1：使用fetch
async function fetchWithAuth(url, options = {}) {
  const token = localStorage.getItem('access_token');
  
  const response = await fetch(url, {
    ...options,
    headers: {
      ...options.headers,
      'Authorization': `Bearer ${token}`
    }
  });
  
  return response.json();
}

// 方式2：使用axios拦截器
import axios from 'axios';

axios.interceptors.request.use(config => {
  const token = localStorage.getItem('access_token');
  if (token) {
    config.headers['Authorization'] = `Bearer ${token}`;
  }
  return config;
});
```

### 4.3 Token过期处理

```javascript
// Token刷新逻辑
async function refreshToken() {
  const refresh_token = localStorage.getItem('refresh_token');
  
  const response = await fetch('/api/auth/refresh', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ refresh_token })
  });

  const result = await response.json();

  if (result.success) {
    localStorage.setItem('access_token', result.data.access_token);
    return true;
  } else {
    // 刷新失败，跳转到登录页
    logout();
    return false;
  }
}

// 响应拦截器（axios）
axios.interceptors.response.use(
  response => response,
  async error => {
    if (error.response.status === 401) {
      // Token过期，尝试刷新
      const success = await refreshToken();
      if (success) {
        // 重试原请求
        return axios.request(error.config);
      }
    }
    return Promise.reject(error);
  }
);
```

### 4.4 权限判断（前端）

```javascript
// 检查用户是否有某个权限
function hasPermission(permission) {
  const permissions = JSON.parse(localStorage.getItem('permissions') || '[]');
  
  // 管理员拥有所有权限
  if (permissions.includes('admin:all')) {
    return true;
  }
  
  return permissions.includes(permission);
}

// 在Vue中使用
// 在组件中
<button v-if="hasPermission('task:intervene')">人工干预</button>

// 在React中使用
{hasPermission('task:intervene') && (
  <button>人工干预</button>
)}
```

### 4.5 路由守卫

```javascript
// Vue Router示例
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('access_token');
  
  // 需要登录的页面
  if (to.meta.requiresAuth && !token) {
    next('/login');
    return;
  }
  
  // 角色权限验证
  if (to.meta.roles) {
    const user = JSON.parse(localStorage.getItem('user'));
    if (!to.meta.roles.includes(user.role)) {
      next('/403'); // 无权限页面
      return;
    }
  }
  
  next();
});
```

## 六、安全说明

### 6.1 密码安全
- 使用 bcrypt 算法加密密码
- 密码哈希存储在 `password_hash` 字段
- 明文密码永不存储

### 6.2 Token安全
- JWT使用 SECRET_KEY 签名
- 访问令牌有效期：30分钟
- 刷新令牌有效期：7天
- 建议生产环境使用强随机SECRET_KEY

### 6.3 环境变量配置

在 `.env` 文件中配置：
```env
# 安全配置
SECRET_KEY=your-super-secret-key-change-in-production-min-32-chars
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
```

**重要**: 生产环境必须使用强随机SECRET_KEY！

生成强随机密钥：
```bash
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

### 6.4 HTTPS部署
⚠️ **生产环境必须使用HTTPS**，否则Token可能被截获！

## 九、常见问题

### Q1: Token过期后怎么办？
**A**: 使用refresh_token获取新的access_token。前端应在401错误时自动调用刷新接口。

### Q2: 如何实现"记住我"功能？
**A**: 前端可以选择将Token保存在localStorage（持久化）或sessionStorage（会话）。

### Q3: 如何踢出在线用户？
**A**: JWT是无状态的，需要实现Token黑名单机制（通常使用Redis存储）。

### Q4: 如何限制同一账号多设备登录？
**A**: 在Token payload中加入设备标识，登录时检查并使旧Token失效。

### Q5: 忘记密码怎么办？
**A**: 需要实现邮箱验证码或手机验证码重置密码功能（后续开发）。
